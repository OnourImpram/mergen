# Support

## Documentation first

Start with the [README](README.md), the [milestone supervisor guide](docs/MILESTONE-SUPERVISOR.md), and the command help
for the entry point you are using.

```bash
mergen --help
mergen-supervise --help
```

## Bug reports

Use the bug report issue form. Include the Mergen commit or version, Python version, operating system, command, minimal
fixture, observed exit code, expected result, and sanitized output. Remove secrets, personal data, and proprietary
artifacts.

## Feature proposals

Use the feature request form. Explain the milestone pattern, the evidence that would be available, the trust boundary,
the intended host, and why an existing deterministic check or profile cannot express the requirement.

## Usage questions

Open a regular issue only when the documentation and existing issues do not answer the question. Keep one problem per
issue and include a minimal reproducible example.

## Security

Do not report vulnerabilities publicly. Follow [SECURITY.md](SECURITY.md) and use a private security advisory.

## Scope

Support is provided on a best-effort basis for the latest repository state and the supported Python versions. Mergen
cannot provide professional legal, clinical, financial, security certification, or approval of a regulated artifact.
