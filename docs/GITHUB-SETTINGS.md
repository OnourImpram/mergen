# GitHub repository settings

The public repository should use the following administration settings.

- Description: Independent milestone verification for agentic and human workflows. Local first, fail closed, evidence aware, and host neutral.
- Topics: verification, milestone-verification, ai-agents, quality-assurance, provenance, governance, ci, adversarial-testing, python.
- Default branch: main.
- Preferred merge method: squash.
- Delete merged branches.
- Protect main from force pushes and deletion.
- Require pull requests, successful CI and security checks, conversation resolution, and an approving review for high-trust changes.

This file records the intended administration baseline. The effective settings are controlled by GitHub repository administration and must be inspected there.
